# 根目录清理方案

## 问题

当前根目录有 **100+ 个文件**，包含大量历史文档、临时文件和测试脚本，导致：
- 难以找到核心文档
- 新用户不知道从哪里开始
- 维护困难
- 看起来不专业

## 解决方案

### 1. 文件分类

将文件分为以下类别：

#### 保留在根目录（核心文件）
- `README.md` / `README.zh-CN.md` - 项目主文档
- `CHANGELOG.md` - 变更日志
- `LICENSE` / `PRIVACY.md` - 许可和隐私
- `DEVELOPMENT.md` / `AGENTS.md` - 开发文档
- `DEPLOY.md` / `TROUBLESHOOTING.md` - 部署和故障排查
- `QUICK_FIX_GUIDE.md` / `SERVER_COMMANDS.md` - 快速参考
- 配置文件（`package.json`、`tsconfig.json` 等）
- Docker 文件（`Dockerfile`、`docker-compose.yml`）

#### 移动到 `docs/archive/`（历史文档）
- API 更新文档（`API_UPDATE_*.md`、`*_SUMMARY.md`）
- 功能完成文档（`*功能完成.md`、`发布成功.md`）
- 修复文档（`*_FIX_*.md`、`修复完成清单.md`）
- 部署历史（`DEPLOYMENT_SUMMARY.md`、`FINAL_DEPLOYMENT.md`）
- 故障排查历史（`DEBUG_*.md`、`FIX_*.md`）
- 发布记录（`PUBLISH_*.md`、`NPM_注册指南.md`）
- 实现报告（`IMPLEMENTATION_REPORT.md`、`PORT_CHANGE_SUMMARY.md`）
- 其他临时文档

#### 移动到 `scripts/`（脚本文件）
- 所有 `.sh` 脚本
- 所有 `.bat` 脚本
- 所有 `.py` 脚本

#### 移动到 `test/`（测试文件）
- 所有 `test-*.js` 文件
- 测试相关脚本

### 3. 清理步骤

#### 自动清理（推荐）

**Linux/macOS:**
```bash
bash scripts/cleanup-root.sh
```

**Windows:**
```cmd
scripts\cleanup-root.bat
```

#### 手动清理

如果需要手动清理：

```bash
# 1. 创建归档目录
mkdir -p docs/archive/{api-updates,features,fixes,deployment,troubleshooting,publish,implementation,misc}

# 2. 移动 API 更新文档
mv API_*.md docs/archive/api-updates/
mv *_SUMMARY.md docs/archive/api-updates/

# 3. 移动功能完成文档
mv *功能完成.md docs/archive/features/
mv 发布*.md docs/archive/features/

# 4. 移动修复文档
mv *FIX*.md docs/archive/fixes/
mv 修复*.md docs/archive/fixes/

# 5. 移动部署文档
mv DEPLOYMENT*.md docs/archive/deployment/
mv OPTIMIZATION*.md docs/archive/deployment/

# 6. 移动故障排查文档
mv DEBUG*.md docs/archive/troubleshooting/
mv FIX_*.md docs/archive/troubleshooting/

# 7. 移动发布文档
mv PUBLISH*.md docs/archive/publish/
mv NPM*.md docs/archive/publish/

# 8. 移动实现报告
mv IMPLEMENTATION*.md docs/archive/implementation/
mv PORT_CHANGE*.md docs/archive/implementation/

# 9. 移动其他文档
mv CHROME_PATH*.md docs/archive/misc/
mv 常驻*.md docs/archive/misc/

# 10. 移动脚本
mv *.sh scripts/
mv *.bat scripts/
mv *.py scripts/

# 11. 移动测试文件
mv test-*.js test/
```

### 4. 清理效果

#### 清理前

## 维护指南

### 1. 新文档放置原则

- **临时文档**：根目录（完成后移到归档）
- **永久文档**：`docs/` 目录
- **历史文档**：`docs/archive/` 目录

### 2. 定期清理

建议每个版本发布后清理一次：

```bash
# 1. 移动完成的功能文档
mv *功能完成.md docs/archive/features/

# 2. 移动修复文档
mv *修复*.md docs/archive/fixes/

# 3. 移动临时脚本
mv temp-*.sh scripts/

# 4. 删除过时文件
rm -f old-*.md
```

### 3. 文档更新

- 保持 README 简洁明了
- 详细内容放在 `docs/` 目录
- 及时更新 CHANGELOG
- 归档历史文档

## 优点
