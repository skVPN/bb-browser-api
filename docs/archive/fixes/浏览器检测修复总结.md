# 浏览器检测修复总结

## 问题描述

用户在 Docker 容器中运行 `bb-browser-api daemon start` 时遇到错误：

```
bb-browser: Cannot find a Chromium-based browser.
Please do one of the following:
  1. Install Google Chrome, Edge, or Brave
  2. Start Chrome with: google-chrome --remote-debugging-port=19825
  3. Set BB_BROWSER_CDP_URL=http://host:port
  4. Use: bb-browser-api daemon start --cdp-url http://localhost:9222
```

但实际上：
- 容器中已安装 Chromium（`/usr/bin/chromium`）
- `which google-chrome` 返回 `/usr/bin/google-chrome`（可能是符号链接）
- CDP 端口 9222 正常运行

## 根本原因

`packages/cli/src/cdp-discovery.ts` 中的 `findBrowserExecutable()` 函数在 Linux 平台上的检测逻辑有问题：

1. **原逻辑**：先用 `which` 命令查找，失败后才检查绝对路径
2. **问题**：在某些 Docker 环境中，`which` 命令可能因为各种原因失败
3. **结果**：即使浏览器已安装在标准路径，也检测不到

## 修复方案

### 1. 调整检测顺序

**修改前**：
```typescript
// 先用 which 命令
const candidates = ["google-chrome", "google-chrome-stable", "chromium-browser", "chromium"];
for (const candidate of candidates) {
  try {
    const resolved = execSync(`which ${candidate}`, ...).trim();
    if (resolved && existsSync(resolved)) {
      return resolved;
    }
  } catch {}
}

// 失败后才检查绝对路径
const commonPaths = ["/usr/bin/google-chrome", ...];
```

**修改后**：
```typescript
// 优先检查绝对路径（Docker 容器中更可靠）
const commonPaths = [
  "/usr/bin/chromium",           // Debian/Ubuntu chromium 包（最优先）
  "/usr/bin/google-chrome",      // Google Chrome 官方包
  "/usr/bin/google-chrome-stable",
  "/usr/bin/chromium-browser",   // 旧版 Ubuntu
  "/snap/bin/chromium",          // Snap 安装
  "/usr/local/bin/google-chrome",
];

for (const candidatePath of commonPaths) {
  if (existsSync(candidatePath)) {
    return candidatePath;
  }
}

// 绝对路径都找不到，才用 which 命令
const candidates = ["chromium", "google-chrome", ...];
```

### 2. 新增诊断工具

创建 `docker/diagnose-browser.sh` 脚本，用于诊断浏览器检测问题：

```bash
docker compose exec bb-browser bash /diagnose-browser.sh
```

检查项目：
- 常见浏览器路径是否存在
- `which` 命令能否找到浏览器
- 浏览器版本信息
- CDP 端口是否可访问
- 相关环境变量

### 3. 新增测试脚本

创建 `test/test-browser-detection.sh`，自动化验证修复：

```bash
docker compose exec bb-browser bash /app/test/test-browser-detection.sh
```

测试内容：
- 浏览器可执行文件检查
- CDP 端口检查
- bb-browser 构建检查
- daemon 功能检查
- 诊断工具检查

## 替代方案

如果修复后仍有问题，可以使用环境变量显式指定 CDP URL：

编辑 `docker-compose.yml`：

```yaml
environment:
  - BB_BROWSER_CDP_URL=http://127.0.0.1:9222
```

这样 daemon 会跳过浏览器检测，直接连接到指定的 CDP 端点。
