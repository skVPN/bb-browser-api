# 🎉 Docker 浏览器检测问题已修复

## 问题

之前在 Docker 容器中启动 daemon 时，可能遇到以下错误：

```
bb-browser: Cannot find a Chromium-based browser.
```

即使容器中已安装 Chromium，仍然检测失败。

## 修复

已优化浏览器检测逻辑，优先检查常见的绝对路径（如 `/usr/bin/chromium`），确保在 Docker 环境中能正确检测到已安装的浏览器。

## 验证修复

```bash
# 运行诊断
docker compose exec bb-browser bash /diagnose-browser.sh

# 运行测试
docker compose exec bb-browser bash /app/test/test-browser-detection.sh

# 检查 daemon
docker compose exec bb-browser bb-browser-api daemon status
```

## 快速修复（如果更新后仍有问题）

编辑 `docker-compose.yml`，添加环境变量：

```yaml
environment:
  - BB_BROWSER_CDP_URL=http://127.0.0.1:9222
```

然后重启：

```bash
docker compose restart
```
