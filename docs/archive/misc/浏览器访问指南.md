# 在浏览器中访问 Chrome 指南

## 🎯 最简单的方案：使用 Selenium + noVNC

### 一键启动

```bash
docker run -d -p 7900:7900 -p 9222:9222 \
  --shm-size="2g" \
  --name chrome-vnc \
  selenium/standalone-chrome:latest
```

### 访问方式

1. **在浏览器中打开**：http://localhost:7900
2. **密码**：`secret`
3. **你会看到一个完整的 Chrome 界面！**

### 连接 bb-browser-api

```bash
bb-browser-api daemon start --cdp-url http://localhost:9222
bb-browser-api open https://github.com
```

**在 VNC 界面中可以实时看到 Chrome 的操作！**

## 📦 Docker Compose 完整配置

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  chrome:
    image: selenium/standalone-chrome:latest
    ports:
      - "7900:7900"  # noVNC Web 界面
      - "9222:9222"  # CDP 端口
    shm_size: 2gb
    environment:
      - SE_VNC_NO_PASSWORD=1  # 不需要密码
      - SE_SCREEN_WIDTH=1920
      - SE_SCREEN_HEIGHT=1080
```

启动：

```bash
docker-compose up -d

# 访问：http://localhost:7900
# 连接：bb-browser-api daemon start --cdp-url http://localhost:9222
```

## 🔒 安全提示

### ⚠️ 不要在公网暴露端口

VNC 和 CDP 端口都没有强认证，不要直接暴露到公网。

### ✅ 安全做法

1. **使用 SSH 隧道**
   ```bash
   ssh -L 7900:localhost:7900 user@server
   ```

2. **配置防火墙**
   ```bash
   # 只允许本地访问
   iptables -A INPUT -p tcp --dport 7900 -s 127.0.0.1 -j ACCEPT
   iptables -A INPUT -p tcp --dport 7900 -j DROP
   ```

3. **使用 VPN**
   - Tailscale
   - WireGuard
   - OpenVPN

## 🆘 常见问题

### Q: VNC 界面黑屏？

A: 等待几秒钟，Chrome 正在启动。

### Q: 无法连接到 CDP？

A: 确认容器正在运行：
```bash
docker ps
curl http://localhost:9222/json/version
```

### Q: 性能不好？

A: 增加 shm_size：
```bash
docker run -d --shm-size="4g" ...
```

### Q: 想要更高分辨率？

A: 设置环境变量：
```bash
docker run -d \
  -e SE_SCREEN_WIDTH=2560 \
  -e SE_SCREEN_HEIGHT=1440 \
  selenium/standalone-chrome
```
