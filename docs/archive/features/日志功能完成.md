# Chrome 启动命令日志功能完成

## 用户需求

用户希望在运行 `bb-browser-api daemon start` 时能看到 Chrome 的完整启动命令，以便：
1. 诊断启动问题
2. 验证启动参数
3. 手动测试启动命令

## 实现方案

### 1. 添加浏览器检测日志

在 `packages/cli/src/cdp-discovery.ts` 的 `findBrowserExecutable()` 函数中添加日志：

```typescript
console.log("[bb-browser] Searching for browser executable on Linux...");
console.log(`[bb-browser] Found browser at: ${candidatePath}`);
console.log("[bb-browser] No browser found in common paths, trying 'which' command...");
console.log(`[bb-browser] Found browser via 'which ${candidate}': ${resolved}`);
console.log("[bb-browser] No browser executable found");
```

### 2. 添加 Chrome 启动命令日志

在 `launchManagedBrowser()` 函数中添加：

```typescript
console.log(`[bb-browser] Launching Chrome: ${executable} ${args.join(" ")}`);
console.error(`[bb-browser] Failed to launch Chrome: ${err instanceof Error ? err.message : String(err)}`);
```

### 3. 添加 CDP 发现日志

在 `packages/cli/src/daemon-manager.ts` 的 `ensureDaemon()` 函数中添加：

```typescript
console.log("[bb-browser] Discovering Chrome CDP endpoint...");
console.log(`[bb-browser] Using custom CDP URL: ${customCdpUrl}`);
console.log(`[bb-browser] CDP endpoint found: ${cdpInfo.host}:${cdpInfo.port}`);
```

### 4. 添加 Daemon 启动日志

```typescript
console.log(`[bb-browser] Starting daemon: ${process.execPath} ${daemonPath} --cdp-host ${cdpInfo.host} --cdp-port ${cdpInfo.port}`);
```

```bash
[bb-browser] Launching Chrome: /usr/bin/chromium --remote-debugging-port=19825 --user-data-dir=/root/.bb-browser/browser/user-data --no-first-run --no-default-browser-check --disable-sync --disable-background-networking --disable-component-update --disable-features=Translate,MediaRouter --disable-session-crashed-bubble --hide-crash-restore-bubble --use-mock-keychain --no-sandbox --disable-setuid-sandbox --disable-dev-shm-usage --disable-gpu about:blank
[bb-browser] Discovering Chrome CDP endpoint...
[bb-browser] CDP endpoint found: 127.0.0.1:19825
[bb-browser] Starting daemon: /usr/bin/node /app/dist/daemon.js --cdp-host 127.0.0.1 --cdp-port 19825

Daemon started successfully
```

### 使用自定义 CDP URL

```bash
$ bb-browser-api daemon start --cdp-url http://127.0.0.1:9222

[bb-browser] Using custom CDP URL: http://127.0.0.1:9222
[bb-browser] CDP endpoint found: 127.0.0.1:9222
[bb-browser] Starting daemon: /usr/bin/node /app/dist/daemon.js --cdp-host 127.0.0.1 --cdp-port 9222

Daemon started successfully
```

### 找不到浏览器

```bash
$ bb-browser-api daemon start

[bb-browser] No browser found in common paths, trying 'which' command...
[bb-browser] No browser executable found

Error: bb-browser: Cannot find a Chromium-based browser.
```

## 使用方法

### 在 Docker 容器中

```bash
# 直接运行
docker compose exec bb-browser bb-browser-api daemon start

# 或进入容器后运行
docker compose exec bb-browser bash
bb-browser-api daemon start
```

### 提取 Chrome 启动命令

从日志中找到以 `[bb-browser] Launching Chrome:` 开头的行，后面就是完整的启动命令。

### 手动测试

```bash
# 复制日志中的命令
/usr/bin/chromium --remote-debugging-port=19825 ... about:blank &

# 等待启动
sleep 5

# 测试 CDP 端口
curl http://127.0.0.1:19825/json/version
```

## 优点

1. **透明性**：用户可以清楚看到每一步操作
2. **可调试性**：出问题时可以快速定位
3. **可复现性**：可以手动复现启动过程
4. **教育性**：帮助用户理解工作原理

## 注意事项

1. 日志输出到 stdout，不影响 JSON 输出
2. 错误日志输出到 stderr
3. 日志格式统一：`[bb-browser] 消息内容`
4. 不影响现有功能和 API
